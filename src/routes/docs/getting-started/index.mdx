# Начало работы с Qwik

Qwik - это новый вид фреймворка, который является [возобновляемым](./../concepts/resumable/index.mdx) (без JS и гидратации), современным и [знакомым для разработчиков на React](./../cheat/qwik-react/index.mdx).

- [Stackblitz Qwik + Vite](https://qwik.new)
- [Песочница с примерами](/examples/reactivity/counter/)

## Подготовка

- [Node.js v16.8](https://nodejs.org/en/download/) или выше
- Ваша любимая IDE (рекомендуется [VSCode](https://code.visualstudio.com/))
- Начните [думать как Qwik](./../think-qwik/index.mdx)

## Создание приложения с помощью CLI

Первый шаг - создание приложения. Qwik поставляется с CLI (CLI - Command Line Interface - интерфейс командной строки), который позволяет вам создать базовый рабочий скелет приложения. Мы будем использовать CLI для создания стартового приложения, чтобы вы могли быстрее познакомиться с ним. Qwik поддерживает NPM, yarn и pnpm.

### Запуск Qwik CLI в терминале

Выберите предпочитаемый менеджер пакетов и выполните одну из следующих команд:

```shell
npm create qwik@latest
pnpm create qwik@latest
yarn create qwik
```

CLI проведёт вас через интерактивное меню, чтобы задать название проекта, выбрать один из стартовых шаблонов, и спросит, хотите ли вы установить зависимости.

## Написание компонентов Qwik

Компоненты - это основные строительные блоки приложений Qwik.

[Компоненты Qwik](../components/overview/index.mdx) должны быть знакомы многим UI-разработчикам:

```tsx
import { component$, useSignal } from '@builder.io/qwik';

export const MyCmp = component$((props: MyCmpProps) => {
  // Объявление локального состояния.
  const count = useSignal(0);

  // Возврат JSX.
  return (
    <>
      <h3>Привет, {props.name}</h3>
      <p>Вы нажали {count.value} раз</p>
      <button
        onClick$={() => {
          // Это обновит локальное состояние и вызовет повторный рендер.
          // Реактивность лежит в основе Qwik!
          count.value++;
        }}
      >
        Увеличить
      </button>
    </>
  );
});
```

Компоненты Qwik уникальны в этом:

- Компоненты Qwik автоматически разбиваются [оптимизатором](../advanced/optimizer/index.mdx) на лениво загружаемые фрагменты;
- Они являются [возобновляемыми](../../concepts/resumable/index.mdx) (компонент может быть создан на сервере и продолжать выполняться на клиенте);
- Они являются [реактивными](../../concepts/reactivity/index.mdx) и рендерятся независимо от других компонентов на странице. См. описание [рендера](../../components/rendering/index.mdx).

Узнайте больше о ключевых концепциях написания [компонентов Qwik](../components/overview/index.mdx).
