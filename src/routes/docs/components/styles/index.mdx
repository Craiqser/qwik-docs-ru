## Глобальные стили

Многие приложения используют глобальные стили для выполнения сброса стилей браузера и/или определения глобальных стилей. Это нормальная практика, но её не рекомендуется использовать для стилизации компонентов. Qwik оптимизирован для того, чтобы браузер загружал только те стили, которые необходимы для текущего представления. Если вы используете глобальные стили, то они будут загружены при первой загрузке, даже если они не нужны на текущей странице.

```tsx
import './global.css'
```

## Встраиваемые критические стили

Чтобы сократить время блокировки первой отрисовки, Qwik может встраивать критические стили текущего представления. Это означает, что браузеру не нужно будет загружать стили для текущего представления. Чтобы указать Qwik на встраивание стилей, просто добавьте к импорту параметр запроса `?inline`.

```tsx
import from './my-component.css?inline'
```

## Почему не встроить стили с помощью тега `<style>`?

Простейший способ убедиться, что в компонент загружены корректные стили, заключается в том, чтобы встроить информацию о стилях в компонент следующим образом:

```typescript
export const MyComponent = () => {
  return (
    <>
      <style>.my-class { color: red; }</style>
      My Component
    </>
  );
}
```

Проблема этого подхода заключается в том, что мы будем загружать стили дважды.

1. Стили вставляются в HTML как часть процесса SSR.
2. Когда компонент меняется и нужно сделать ререндер, стили загружаются снова, поскольку они встроены.

Необходимо загружать стили независимо от компонента. Для этого и существует метод `useStyles$()`. Есть два сценария:

1. Компонент рендерится на сервере, и стили вставляются в секцию `<head>`, как часть SSR.
  - Добавление нового экземпляра компонента в приложение не требует загрузки стилей, так как они уже подключены в процессе SSR.
2. Компонент рендерится на клиенте в первый раз. В этом случае новый компонент не будет иметь стилей в `<head>`, так как компонент не был частью SSR.
  - Добавление нового компонента, который не был частью SSR, требует загрузки и вставки стилей в `<head>`.

## `useStyles$()`

Это лениво-загружаемая ссылка на стили компонента.

Стили компонентов позволяют Qwik лениво загружать информацию о стиле для компонента только тогда, когда это необходимо (и избегать двойной загрузки в случае гидратации SSR).

```tsx
import {useStyles$} from '@builder.io/qwik';
import styles from './code-block.css';

export const CmpStyles = component$(() => {
  useStyles$(styles);
  return <span class="my-text">Какой-то текст</span>;
});
```

```css
// code-block.css
.my-text {
  color: red;
}
```

## `useStylesScope$`

В предыдущем разделе мы обсуждали, как можно лениво загружать стили по мере необходимости с помощью хука `useStyles$()`.
Стили браузера являются глобальными и применяются ко всем элементам DOM, по этой причине Qwik также предоставляет способ загрузки стилей, специфичных для определённого компонента.
Это достигается путем создания уникального класса для каждого компонента и последующей вставки этого уникального идентификатора класса в таблицу стилей.

```tsx
import {useStylesScoped$} from '@builder.io/qwik';
import styles from './code-block.css';

export const CmpStyles = component$(() => {
  useStylesScoped$(styles);
  return <span class="my-text">Какой-то текст</span>;
});
```

```css
// code-block.css
.my-text {
  color: blue;
}
```

## Preprocessors

### SASS / SCSS

Qwik прекрасно работает с SCSS и SASS без каких-либо дополнительных библиотек. Просто убедитесь, что ваши файлы имеют расширение `.scss` или `.sass`.

### Tailwind

Чтобы использовать в своём приложении Tailwind, следуйте этим [инструкциям](../../../qwikcity/integrations/tailwind).
