Одна из сверхспособностей Qwik заключается в его функциях ленивой загрузки.

Qwik лениво загружает:

- **Независимо от иерархии**: Компоненты могут быть загружены не по порядку. Например, код дочернего компонента может загружаться раньше его родительского кода.
- **На основе взаимодействия**: Загрузка кода откладывается до тех пор, пока пользователь не взаимодействует с компонентом.
- **Больше, чем просто компоненты**: Qwik лениво загружает любое замыкание, включая компоненты, слушателей событий, эффекты и поведение.

`$` помечает замыкание как лениво загружаемое. Например, метод `component$()` делает компонент доступным для ленивой загрузки. Когда вы видите `$` в коде Qwik, вы пересекаете границу отложенной загрузки и должны знать о специальных правилах:

- любые переменные с лексической областью действия должны быть объявлены как `const`;
- захваченная переменная/символ должны быть либо:
  - сериализуемы;
  - импортируемы (либо из другого файла с помощью `import`, либо из этого файла с помощью `export`).

Если вы хотите, чтобы компонент загружался вместе с другим компонентом, вы создаёте встроенный компонент. Встроенные компоненты загружаются как часть родительского компонента, это эквивалентно тому, как большинство других фреймворков работают с компонентами.

В этом примере для вас подготовлены компоненты `<App>` и `<Greeter>`. Компонент `<Greeter />` объявляется с помощью метода `component$()` и является компонентом Qwik. Удалите `component$()`, чтобы преобразовать `<Greeter>` во встроенный компонент.

Откройте вкладку `Symbols` и обратите внимание, что компонент `<Greeter />` больше не является независимым, а включен как часть компонента `<App>`.
