Используйте `useMount$()` для выполнения кода в момент, когда компонент монтируется в дерево рендеринга (вы можете представлять что `useMount$()` - это момент создания компонента).

## Жизненный цикл компонента и серверный рендер (SSR)

Qwik является возобновляемым. Возобновляемость означает, что приложение запускается на сервере, а затем передаётся клиенту. На клиенте приложение продолжает выполнение с того места, где остановилось. Следствием этого является то, что компонент может быть создан на сервере и уничтожен на клиенте. Это означает, что метод компонента `useMount$()` может выполняться на сервере, но метод `useCleanup$()` может выполняться на клиенте.

## Использование `useMount$()`

`useMount$()` - это хук, который выполняет обратный вызов когда компонент монтируется в дерево рендеринга. Функция `useMount$()` может быть асинхронной. `useMount$()` задерживает рендер компонента до завершения выполнения функции обратного вызова. Типичное использование `useMount$()` - это получение данных, необходимых для начального рендеринга.

## Вариации `useMount$()`

Хук `useMount$()` также имеет серверную версию `useServerMount$()`, которая выполняется при монтировании компонента в серверной среде. Это полезно, поскольку сервер часто имеет другое API для получения данных.

Используйте `useMount$()`, если код, который должен быть выполнен, идентичен на сервере и клиенте.

## Импорты, требуемые только серверу

Поскольку хуки `useServerMount$()` имеют в своем названии `$`, они подвержены ленивой загрузке. Ленивая загрузка означает, что функция переносится оптимизатором в новый файл. При перемещении функции оптимизатор также перемещает вместе с ней любые импорты. Это означает, что импорты, предназначенные только для сервера, можно безопасно использовать как часть хука `useServerMount$()`, поскольку они будут удалены оптимизатором.
