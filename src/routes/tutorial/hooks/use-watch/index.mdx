Используйте [`useWatch$()`](/docs/components/lifecycle/index.mdx#usewatch) для выполнения функции перед начальным рендером и при каждом изменении отслеживаемых значений. Функция выполняется перед рендером, но она не может задержать рендер, поэтому если [`useWatch$()`](/docs/components/lifecycle/index.mdx#usewatch) является асинхронной, рендер произойдет до того, как [`useWatch$()`](/docs/components/lifecycle/index.mdx#usewatch) будет полностью выполнена.

## Отслеживание изменений в хранилище

Хук [`useWatch$()`](/docs/components/lifecycle/index.mdx#usewatch) получает функцию `track()`, используемую для настройки подписок, которые будут автоматически повторять хук [`useWatch$()`](/docs/components/lifecycle/index.mdx#usewatch). Набор свойств `track()` сбрасывается при каждом выполнении [`useWatch$()`](/docs/components/lifecycle/index.mdx#usewatch), поэтому важно всегда использовать `track()` для настройки подписок заново. Из-за этого [`useWatch$()`](/docs/components/lifecycle/index.mdx#usewatch) может подписываться на разные свойства с течением времени.

## Очистка

Хук [`useWatch$()`](/docs/components/lifecycle/index.mdx#usewatch) может вернуть функцию обратного вызова для очистки. Это полезно для очистки любых ресурсов перед новым выполнением.
