Используйте контекст для передачи данных дочерним компонентам без их явной передачи через компоненты (`prop drilling`). Контекст полезен для обмена данными, которые необходимы всем компонентам приложения. Например, информация о стиле, состоянии приложения или текущем пользователе, вошедшем в систему.

Для использования контекста вам понадобятся три вещи:

- `createContextId()`: Создаёт сериализуемый идентификатор для контекста. Убедитесь, что этот идентификатор уникален в вашем приложении.
- `useContextProvider()`: Вызов в родительском компоненте приводит к публикации значения контекста. Все дочерние компоненты, являющиеся потомками этого компонента (и внуками), смогут получить контекст.
- [`useContext()`](/docs/components/context/index.mdx#usecontext): Для получения контекста и использования его в любом дочернем компоненте.

В этом примере мы хотим передать `TodosStore` компоненту `<Items>`. Обновите код, чтобы использовать [`useContext()`](/docs/components/context/index.mdx#usecontext) для получения значения.

Контексты обычно являются хранилищами, и поэтому они должны быть сериализуемыми.
