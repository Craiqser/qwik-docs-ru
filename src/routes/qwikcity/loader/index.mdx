# `loader$()`

Загрузчики позволяют передавать данные с сервера компонентам Qwik. По этой причине загрузчики Qwik всегда следует понимать в контексте **загрузки данных на сервере, которые впоследствии используются компонентом Qwik**. Если вы хотите создать конечную точку RESTful, то ознакомьтесь с руководством по [конечным точкам](/qwikcity/endpoints/index.mdx).

Они ведут себя как серверные функции RPC, которые могут вызываться компонентами Qwik во время рендера.

## Объявление загрузчика

Загрузчики могут быть объявлены только внутри папки `src/routes`, в файле `layout.tsx` или `index.tsx`, и они ДОЛЖНЫ быть экспортированы.

```tsx
// src/routes/layout.tsx
import { loader$ } from '@builder.io/qwik-city';

export const getProductData = loader$(() => {
  return {
    product: {
      name: 'Qwik City',
      price: 100,
    },
  };
});
```

> Загрузчики не являются конечными точками, это внутренний канал связи между сервером и компонентами Qwik. Кроме того, загрузчики должны возвращать сериализуемое JSON-значение.

## Использование загрузчика

Загрузчики могут использоваться любым компонентом приложения, если загрузчик объявлен в файле `layout.tsx` или `index.tsx`, который является частью существующего маршрута.

```tsx
// src/routes/index.tsx
import { loader$ } from '@builder.io/qwik-city';
import { component$ } from '@builder.io/qwik';

export const useGetServerTime = loader$(() => {
  return {
    time: Date.now();
  }
});

export default component$(() => {
  // Retrieve a reactive signal of the loader data
  const signal = useGetServerTime(); // Signal<{time: number}>
  return (
    <div>
      Время на сервере: {signal.value.time}
    </div>
  );
});
```

> `.use()` запрашивает данные загрузчика, но не выполняет загрузчик несколько раз. Загрузчики выполняются в начале запроса, чтобы обеспечить низкую задержку. По этой причине они разрешены только в папке `src/routes`, в файле `layout.tsx` или `index.tsx`, и они ДОЛЖНЫ быть экспортированы.

## Несколько загрузчиков

В приложении разрешено использовать несколько загрузчиков, и их можно использовать в любом компоненте Qwik. **Вы даже можете объявить несколько загрузчиков в одном файле**.

**File: `src/routes/layout.tsx`**

```tsx
import { component$ } from '@builder.io/qwik';
import { loader$ } from '@builder.io/qwik-city';
import { Footer } from '../components/footer.tsx';

export const useGetServerTime = loader$(() => {
  return {
    time: Date.now();
  }
});

export default component$(() => {
  const signal = useGetServerTime();
  return (
    <main>
      <Slot>
      <Footer />
    </main>
  );
});
```

**File: `src/components/footer.tsx`**

```tsx
import { component$ } from '@builder.io/qwik';

// Импорт загрузчика из макета
import { useGetServerTime } from '../routes/layout.tsx';

export const Footer = component$(() => {
  // Использование данных загрузчика
  const signal = useGetServerTime();
  return <footer>Время на сервере: {signal.value.time}</footer>;
});
```

**File: `src/routes/admin/index.tsx`**

```tsx
import { component$ } from '@builder.io/qwik';
import { loader$ } from '@builder.io/qwik-city';

export const useIsLogged = loader$(({cookies}) => {
  return {
    logged: checkCookies(cookies);
  }
});

export const useGetCurrentUser = loader$(({cookies}) => {
  return {
    user: currentUserFromCookies(cookies);
  }
});

export default component$(() => {
  const logged = useIsLogged();
  const user = useGetCurrentUser();
  return (
    <section>
      <h1>Администратор</h1>
      {logged.value.logged ? (
        <p>Привет, {user.value.user.name}</p>
      ) : (
        <p>Вы не авторизованы</p>
      )}
    </section>
  );
});
```

## Контекст загрузчика

Как и обработчики запросов, такие как `onRequest` и `onGet`, загрузчики имеют доступ к объекту `RequestEvent`, который содержит информацию о текущем запросе.

Эта информация пригодится, когда загрузчику необходимо вернуть данные по условию на основе запроса или ему необходимо вручную переопределить статус ответа, заголовки или тело.

```tsx
export const getServerTime = loader$((ev) => {
  console.log('Request headers:', ev.headers);
  console.log('Request cookies:', ev.cookies);
  console.log('Request url:', ev.url);
  console.log('Request method:', ev.method);
  console.log('Request params:', ev.params);

  return {
    time: Date.now();
  }
});
```

## Когда использовать загрузчик?

Загрузчик следует использовать, когда вам нужно предоставить некоторые серверные данные для ваших компонентов Qwik. Например, если вам нужно получить некоторые данные из базы данных или API, вы можете использовать для этого загрузчик.

Вы не должны использовать загрузчик для создания REST API, для этого вам лучше использовать [Конечную точку](/qwikcity/endpoints/index.mdx), которая позволяет вам иметь жесткий контроль над заголовками и телом ответа.
