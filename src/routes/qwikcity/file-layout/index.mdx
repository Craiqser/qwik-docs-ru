# Расположение файлов

Основная задача Qwik City - предоставить простой, интуитивно понятный и эффективный способ создания контента для вашего сайта. Qwik City достигает этого благодаря маршрутизации на основе файлов. В двух словах, расположение ваших файлов определяет структуру вашего сайта.

## Каталог `routes`

Вся информация о маршрутах, основанная на файлах, помещается в каталог `src/routes`. Это единственный каталог, где соглашения об именовании файлов имеют значение для вашего сайта (во всех остальных папках названия не имеют никакого значения для Qwik City).

Папка `src` обычно также содержит папку `components`, но их название и расположение не имеют значения для Qwik City.

## Пример директории `routes`

```
src/
├── components/                # Поместите сюда свои переиспользуемые компоненты.
│    └── counter.tsx
└── routes/                    # Поместите сюда компоненты, специфичные для вашего маршрута.
    ├── docs/
    │   ├── overview/
    │   │   └── index.mdx      # https://example.com/docs/overview
    │   └── index.mdx          # https://example.com/docs
    ├── index.tsx              # https://example.com/
    └── layout.tsx             # Все вложенные страницы используют этот макет.
```

## Пример компонента

Qwik City ожидает, что файлы `index.tsx`, находящиеся в папке `src/routes`, имеют экспорт `default`.

```tsx
import { component$ } from '@builder.io/qwik';

export default component$(() => {
  return <>Привет, мир!</>;
});
```
