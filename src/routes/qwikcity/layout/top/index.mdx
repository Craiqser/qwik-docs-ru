# Макет верхнего уровня

По умолчанию страница будет создавать свой стек макетов, поднимаясь вверх по каждой директории, пока не доберётся до директории `src/routes`. Если в какой-то момент требуется прекратить подъём по каталогам, вы можете воспользоваться макетом "верхнего уровня".

Под макетом верхнего уровня понимается то, что он всегда находится на самом верху стека макетов, и страница не продолжает собирать родительские макеты, даже если они есть в родительских каталогах.

Макет верхнего уровня задаётся путём завершения имени файла макета символом `!`: например, `layout!.tsx`.

```
src/
└── routes/
    ├── layout.tsx              # Корневой макет
    └── docs/
        ├── layout!.tsx         # Макет документов
        ├── index.tsx           # https://example.com/docs
        └── api/
            └── index.tsx       # https://example.com/docs/api
```

В приведённом выше примере в родительских каталогах страницы `src/routes/docs/api/index.tsx` находятся два макета. По умолчанию страница `/docs/api` будет вложена в оба макета (корневой и макет документов).

Однако, поскольку имя файла `src/routes/docs/layout!.tsx` заканчивается на `!`, то этот макет становится самым верхним макетом, а корневой макет `src/routes/layout.tsx` перестаёт являться частью стека макетов.
