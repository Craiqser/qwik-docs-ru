# Группировка макетов

Маршруты общего назначения часто помещают в каталоги, чтобы они могли совместно использовать макеты, и чтобы связанные исходные файлы были логически сгруппированы друг с другом. Однако может возникнуть желание, чтобы каталог, который использовался для группировки похожих файлов и совместного использования макетов, был исключен из общедоступных URL-адресов. Здесь поможет "группировка" макетов (так называемый макет маршрута "без пути").

Если к любому имени каталога добавить два символа подчёркивания "__", то само имя каталога не будет включено в URL-путь.

Например, предположим, что все маршруты учётных записей в приложении расположены в одном каталоге `/accounts/`, однако он может быть удалён из URL-адреса для более чистых и кратких URL-адресов. В приведённом ниже примере обратите внимание на то, что пути находятся в каталоге `src/routes/__accounts/`, однако URL-адреса исключают `__accounts/`.

```
src/
└── routes/
    └── __accounts/                    # Notice the double underscore prefix
        ├── layout.tsx                 # Shared account layout
        └── profile/
            └── index.tsx              # https://example.com/profile
        └── settings/
            └── index.tsx              # https://example.com/settings
```
