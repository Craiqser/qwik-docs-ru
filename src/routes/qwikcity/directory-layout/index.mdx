# Структура файлов

Основная задача Qwik City - предоставить простой, интуитивно понятный и эффективный способ создания контента для вашего сайта. Qwik City достигает этого благодаря маршрутизации на основе каталогов. В двух словах, расположение ваших каталогов определяет структуру вашего сайта.

## Каталог `routes`

Вся информация о маршрутах, основанная на каталогах, содержится в каталоге `src/routes`. Это единственный каталог, где соглашения об именовании вложенных каталогов и файлов имеют значение для вашего сайта (во всех остальных каталогах названия не имеют для Qwik City никакого значения).

Например, каталог `src` также обычно содержит каталог `components`, но название и расположение не имеют для Qwik City никакого значения.

## Пример каталога `routes`

```
──src/
  ├─components/          # Поместите сюда свои переиспользуемые компоненты.
  │ └─counter.tsx
  └─routes/              # Поместите сюда компоненты, специфичные для маршрута.
    ├─docs/
    │ ├─overview/
    │ │ └─index.mdx      # https://example.com/docs/overview
    │ └─index.mdx        # https://example.com/docs
    ├─index.tsx          # https://example.com/
    └─layout.tsx         # Все вложенные страницы используют этот макет.
```

## Пример компонента

Qwik City ожидает, что файлы `index.tsx`, находящиеся в каталоге `src/routes`, имеют экспорт по умолчанию.

```typescript
import { component$ } from '@builder.io/qwik';

export default component$(() => {
  return <>Hello World!</>;
});
```
